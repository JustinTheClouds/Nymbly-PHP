{"Templating"}

Nymbly comes with a very minimal templating system in place. It supports only 5 template patterns.

Variables

Easily output any assigned variables inside of your template files.

Assigning variables

From within your controller call

$this->assign('somevarname', 'some value');

To use the variable in a teamplate file

{$somevarname}

Translations

The translation pattern allowed you to easily write out text that will be checked for translations automatically.

Examples
{'Just write the text you want within quotes'} {"Double quotes work the same"} {"But what if text requires a ' inside of it?"} {'Then use the opposite ".'} {'What if it needs both ' and "?'}

Links

The link pattern is useful because it will automatically append the baseurl infront of the link. The helps when working locally. The link pattern must begin with a / and be used without quotes or else it will be treated as a tranlation.

{/demo/templating}

will return {/demo/templating}

Controllers

The controller pattern is the most powerful. This will allow you to embed other controllers directly into views.

The syntax is

{option.task.action}

For example

{demo.layouts.nav}

Will try to load the controllers of demo/layouts/nav. If they exist, the view will be outputted in place. They make templating of reusable views very powerful.

The task and action can be ommitted as well and are not required.

{demo}

Example, embedding the demo nav bar

{demo.layout.nav}

Result:

{demo.layouts.nav}
